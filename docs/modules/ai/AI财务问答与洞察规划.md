# AI 财务问答与洞察规划

## 定位

AI 财务问答与洞察是 M6 候选能力，用于让用户通过自然语言询问账本数据，并得到基于真实查询结果的分析答复。

示例问题：

```txt
这周消费了多少钱，主要是用在什么地方？
```

该能力不同于 M4 AI 文本记账。M4 负责把文本转换为候选流水；M6 负责理解用户问题、查询已授权账本数据并生成分析答复。

## 当前状态

- 尚未实现对话式 AI 问答接口。
- 尚未接入真实 LLM provider 或 API Key。
- 已具备后续实现所需的部分基础能力：Ledger Policy、流水查询、基础统计 API、AI 任务表和 NestJS 调 FastAPI 的内部边界。

## 核心边界

- 前端、后台 Web 和移动端只能调用 NestJS 对外 API，不直接调用 FastAPI。
- NestJS 负责认证、账本权限、交易可见性、工具执行、真实数据查询和任务状态保存。
- FastAPI/LLM 只负责理解问题、选择受控工具和组织自然语言答复。
- FastAPI/LLM 不允许直接访问数据库，不允许绕过 Ledger Policy，不允许自行编造金额、分类占比、成员消费或时间范围。
- 系统管理员默认只能查看脱敏 AI 任务摘要，不展示用户原始问题、完整模型输出或完整账本明细。

## 推荐流程

```txt
用户提问
  -> POST /ledgers/:ledgerId/ai/insights/chat
  -> NestJS 校验用户和账本权限
  -> NestJS 创建 ai_task: insight
  -> NestJS 准备可用工具列表和最小必要上下文
  -> FastAPI/LLM 判断需要哪些工具
  -> NestJS 执行统计或流水查询工具
  -> FastAPI/LLM 基于查询结果生成答复
  -> NestJS 保存脱敏任务摘要、结构化引用数据和答复
  -> 前端展示答复和数据来源
```

## 初始工具候选

- 查询指定时间范围内的收入、支出和结余汇总。
- 查询指定时间范围内的分类支出占比。
- 查询账户余额汇总。
- 查询成员消费汇总。
- 查询指定时间范围内的流水摘要，默认只返回必要字段和分页样本。

这些工具应优先复用现有统计 API 和流水查询能力；如现有接口口径不足，再由 NestJS 新增内部 query service。

## 后续设计要点

- 需要新增独立设计文档和实施计划后再实现。
- 需要定义 `ai_tasks.type = insight` 下的任务详情、答复结构和引用数据结构。
- 需要决定真实 LLM provider、API Key 管理方式、超时、重试、熔断和成本统计。
- 需要为“本周”“本月”“上个月”等相对时间表达定义账本 timezone 下的解析规则。
- 需要为私密流水、共享流水、家庭成员数据和系统管理员排查场景补充隐私规则。
- 需要提供可重复验证的集成测试，确保金额来自查询结果而不是模型自由生成。
