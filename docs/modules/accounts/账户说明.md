# 账户说明

## 功能目标

账户模块提供账本下现金、银行卡、支付宝、微信、信用卡和其他账户的基础管理能力。账户必须归属于 `ledger`，并支持共享账户和私密账户，为收入、支出、转账和后续账户统计提供基础数据。

## 业务规则

- 所有账户必须包含 `ledger_id`，不能建模为只归属于用户。
- `visibility = ledger` 的账户对账本 active 成员可见。
- `visibility = private` 的账户仅 `owner_id` 可见。
- 创建私密账户时必须写入 `owner_id`，默认使用当前用户。
- 私密账户下创建的流水默认 `visibility = private`。
- 账户删除第一版采用 `archived_at` 软删除，不物理删除历史数据。
- `current_balance` 在 M1.5 由账户创建和账户更新接口维护；流水创建暂不自动变更账户余额，后续通过余额重算和流水联动统一补齐。
- 金额字段使用 Decimal 字符串入参，Prisma Decimal 存储；接口校验按 `Decimal(18, 2)` 约束，避免 float 误差。
- Controller 只处理 HTTP 入参和响应，业务规则放在 Service，数据访问放在 Repository/Prisma 层，权限判断集中在 Ledger Policy 层。

## 数据模型

`accounts`

```txt
id
ledger_id
name
type                 cash | bank_card | alipay | wechat | credit_card | other
currency
initial_balance
current_balance
visibility           ledger | private
owner_id
sort_order
created_at
updated_at
archived_at
```

建议索引：

```txt
accounts(ledger_id, visibility)
accounts(ledger_id, archived_at)
accounts(owner_id)
```

## 接口说明

- `GET /ledgers/:ledgerId/accounts`：查询当前用户可见账户列表，返回共享账户和当前用户自己的私密账户。
- `POST /ledgers/:ledgerId/accounts`：创建账户，M1.5 默认 owner/admin 可创建共享或私密账户。
- `PATCH /accounts/:accountId`：更新账户名称、类型、币种、余额、可见性和排序。
- `DELETE /accounts/:accountId`：软删除账户。

## 权限规则

- 访问账本账户列表前必须通过 `canViewLedger(userId, ledgerId)`。
- 创建、更新、删除共享账户需要 `canManageLedger(userId, ledgerId)`。
- `canViewAccount(userId, accountId)`：
  - 账户不存在或已归档时返回 false。
  - 共享账户要求用户是账本 active 成员。
  - 私密账户要求 `owner_id = userId`。
- 私密账户的更新和删除必须先通过 `canViewAccount(userId, accountId)`，再由 Policy 确认当前用户是 active 成员且为账户 owner；共享账户由 owner/admin 管理。
- 按 `accountId` 更新或删除不可见账户时，M1.5 返回不泄露资源存在性的 not found 风格响应。

## 异常情况

- 账本不存在或已归档：`LEDGER_NOT_FOUND`。
- 非账本成员访问：`LEDGER_ACCESS_DENIED`。
- 角色能力不足：`MEMBER_ROLE_DENIED`。
- 账户不存在或不可见：`ACCOUNT_NOT_FOUND` 或 `PRIVATE_RESOURCE_DENIED`，M1.5 以不泄露私密资源存在性为原则。
- Decimal 金额格式非法：`VALIDATION_FAILED`。
