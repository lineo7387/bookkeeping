# 基础统计说明

## 功能目标

基础统计模块提供账本内可落地的 MVP 报表 API，帮助用户查看月度收入/支出、分类占比、账户余额和成员消费情况。统计数据必须基于当前用户可见范围生成，不能向其他成员泄露私密流水或私密账户。

## 业务规则

- 所有统计接口必须归属于 `ledger`，路径统一挂在 `/ledgers/:ledgerId/statistics/*`。
- 访问统计前必须通过 Ledger Policy 的 `canViewLedger` 校验，只有账本 active 成员可以查看。
- 统计只包含 `deleted_at = null` 的正式流水。
- 金额使用 Decimal 聚合和字符串输出，不使用 float。
- 月度收入/支出汇总只统计 `income` 和 `expense`，不把 `transfer` 计入收入或支出。
- 分类占比按分类聚合 `income` 或 `expense` 流水；未设置分类的有效流水归入 `categoryId = null`、`categoryName = null`。
- 账户余额统计直接读取当前用户可见账户的 `current_balance`，与 M2 账户余额流水联动结果保持一致。
- 成员消费统计只统计 `expense` 流水，按 `created_by` 聚合。
- 私密流水仅创建者可见并参与自己的统计。
- 私密账户仅 `owner_id` 可见；账户余额统计不返回其他成员的私密账户。
- 共享流水如果关联他人的私密账户，当前数据模型下仍按流水可见性参与流水类统计；账户余额类统计仍隐藏不可见私密账户。

## 查询参数

月度、分类和成员统计支持时间范围：

```txt
occurredFrom: 带 Z 或时区偏移的 ISO 8601 日期时间，可选
occurredTo:   带 Z 或时区偏移的 ISO 8601 日期时间，可选
```

分类占比额外支持：

```txt
type: income | expense，默认 expense
```

账户余额统计第一版不接收时间范围，始终返回当前余额快照。

## 接口说明

- `GET /ledgers/:ledgerId/statistics/monthly`：返回当前用户可见流水在时间范围内的收入、支出和结余。
- `GET /ledgers/:ledgerId/statistics/categories?type=expense`：返回当前用户可见流水按分类聚合后的金额、笔数和占比。
- `GET /ledgers/:ledgerId/statistics/accounts`：返回当前用户可见账户的余额列表和总余额。
- `GET /ledgers/:ledgerId/statistics/members`：返回当前用户可见支出流水按创建成员聚合后的金额和笔数。

## 权限规则

- Controller 使用 `JwtAuthGuard` 和 `RequireLedgerPolicy('view')` 保护所有统计接口。
- Repository 查询必须再次按用户可见范围过滤数据：
  - 流水类统计：`visibility = ledger` 或 `visibility = private AND created_by = currentUserId`。
  - 账户余额统计：`visibility = ledger` 或 `visibility = private AND owner_id = currentUserId`。
- `selected_members` 在第一版不参与可见统计，后续补充成员授权表后再扩展。

## 异常情况

- 未登录：`AUTH_REQUIRED`。
- 非账本 active 成员访问：`LEDGER_ACCESS_DENIED`。
- 查询参数时间不是带时区的 ISO 日期时间：`VALIDATION_FAILED`。
- 分类统计 `type` 非 `income` 或 `expense`：`VALIDATION_FAILED`。
