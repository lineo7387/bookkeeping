# 分类说明

## 功能目标

分类模块提供账本下收入和支出分类管理能力。第一版 API 和数据模型预留 `parent_id` 支持树形分类，但 UI 只开放一层分类，避免早期操作复杂度过高。

## 业务规则

- 所有分类必须包含 `ledger_id`。
- 分类类型只支持 `income` 和 `expense`，转账不使用收支分类。
- `parent_id` 用于预留树形结构；M1.5 创建和更新时允许为空，若传入必须属于同一账本且类型一致。
- `is_system = true` 的系统分类第一版可以由后端种子或未来初始化流程创建，普通接口不得删除系统分类。
- 分类删除第一版采用 `archived_at` 软删除。
- 分类排序使用 `sort_order`，同一账本同一类型下按 `sort_order asc, created_at asc` 返回。
- Controller 只处理 HTTP 入参和响应，Service 负责业务流程，Repository/Prisma 层负责数据访问。

## 数据模型

`categories`

```txt
id
ledger_id
parent_id
type                 income | expense
name
icon
color
is_system
sort_order
created_at
updated_at
archived_at
```

建议索引：

```txt
categories(ledger_id, type)
categories(ledger_id, archived_at)
categories(parent_id)
```

## 接口说明

- `GET /ledgers/:ledgerId/categories`：查询分类列表，可选按 `type` 筛选。
- `POST /ledgers/:ledgerId/categories`：创建分类，owner/admin 可创建。
- `PATCH /categories/:categoryId`：更新分类名称、图标、颜色、排序和父级。
- `DELETE /categories/:categoryId`：软删除分类。

## 权限规则

- 查看分类要求 `canViewLedger(userId, ledgerId)`。
- 创建、更新、删除分类要求 `canManageLedger(userId, ledgerId)`。
- 按 `categoryId` 更新或删除时，Service 先通过用户可见范围查找 active 分类；非成员、已归档或不存在的分类统一返回 `VALIDATION_FAILED`，避免泄露分类是否存在。
- 系统分类禁止通过普通删除接口删除。
- 删除父分类时，M1.5 不级联删除子分类；若存在未归档子分类，Service 返回 `VALIDATION_FAILED`。

## 异常情况

- 账本不存在或已归档：`LEDGER_NOT_FOUND`。
- 非账本成员访问：`LEDGER_ACCESS_DENIED`。
- 角色能力不足：`MEMBER_ROLE_DENIED`。
- 分类不存在或已归档：`VALIDATION_FAILED` 或后续扩展独立错误码。
- 父分类不属于同一账本、类型不一致或形成自引用：`VALIDATION_FAILED`。
- 删除系统分类或仍有子分类的分类：`VALIDATION_FAILED`。
