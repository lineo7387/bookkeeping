# 流水说明

## 功能目标

流水模块提供账本下收入、支出和转账的基础闭环。所有流水必须归属于 `ledger`，并记录 `created_by`，支持共享流水、私密流水和未来 selected members 可见性。

## 业务规则

- 所有流水必须包含 `ledger_id` 和 `created_by`。
- 类型支持 `income`、`expense`、`transfer`。
- 金额使用 Decimal，不使用 float；接口金额使用字符串，例如 `"86.00"`。
- 时间统一存 UTC，接口时间必须使用带 `Z` 或时区偏移的完整 ISO 8601 日期时间字符串，避免服务端本地时区导致写入漂移。
- `visibility = ledger` 的流水对账本 active 成员可见。
- `visibility = private` 的流水仅 `created_by` 可见和管理。
- `visibility = selected_members` 在 M1.5 进入枚举和接口约束，但成员选择表可作为后续扩展；第一版不开放创建 selected members 流水。
- 私密账户下创建的流水默认 `visibility = private`。
- 第一版转账使用一条 `type = transfer` 记录，并在 `metadata.transferTargetAccountId` 保存目标账户 ID；Service 封装转账逻辑，避免未来升级双分录模型时影响前端接口。
- M2 起正式流水创建、更新和删除会自动联动账户 `current_balance`：收入增加源账户余额，支出减少源账户余额，转账减少源账户余额并增加目标账户余额。
- 更新流水必须先撤销旧流水余额影响，再应用新流水余额影响；删除流水必须软删除并撤销旧余额影响。
- 流水写入、更新或软删除与账户余额增量更新必须在同一个 Prisma transaction 内完成。
- 转账源账户和目标账户不能相同，避免产生无意义流水和余额净零变动。
- 删除流水采用 `deleted_at` 软删除。
- Controller 只处理 HTTP 入参和响应，Service 负责业务流程，Repository/Prisma 层负责数据访问，权限判断集中在 Ledger Policy 层。

## 接口说明

- `GET /ledgers/:ledgerId/transactions`：查询当前用户可见流水列表，M1.5 支持基础分页、类型、账户、分类和日期范围筛选。
- `POST /ledgers/:ledgerId/transactions`：创建流水，owner/admin/editor 可创建；接口不允许客户端指定 `source`，M1.5 手工创建统一为 `manual`。
- `GET /transactions/:transactionId`：查询单条流水，必须通过 `canViewTransaction`。
- `PATCH /transactions/:transactionId`：更新流水，必须通过 `canUpdateTransaction`。
- `DELETE /transactions/:transactionId`：软删除流水，复用更新权限或单独 `canDeleteTransaction` 扩展点。

## 权限规则

- `canCreateTransaction(userId, ledgerId)`：owner/admin/editor 可创建。
- `canViewTransaction(userId, transactionId)`：
  - 流水不存在或已删除时返回 false。
  - 私密流水仅 `created_by` 可见。
  - 共享流水要求用户是账本 active 成员。
  - selected members 流水 M1.5 暂不开放创建；未来需检查成员授权表。
- `canUpdateTransaction(userId, transactionId)`：
  - 私密流水仅 `created_by` 可管理。
  - owner/admin 可更新共享流水。
  - editor 只能更新自己创建的共享流水。
  - viewer 不可更新。
- 创建流水时必须校验账户可见性；私密账户仅 owner 可使用。
- 创建流水时必须校验分类属于同一账本且类型匹配收入/支出；转账不要求分类。
- 创建转账流水必须提供目标账户；源账户或目标账户任一为私密账户时，流水强制为 `private`。
- 更新转账流水时会重新校验源账户和目标账户；转账改为非转账时会清空转账 metadata。
- 按 `transactionId` 查询、更新和删除时先走 Policy 判断；不可见、已删除或不存在的流水返回不泄露资源存在性的 not found 风格响应。由于共享错误码中暂未定义 `TRANSACTION_NOT_FOUND`，M1.5 使用 HTTP 404 搭配 `VALIDATION_FAILED` 表示流水不存在或不可访问。

## 异常情况

- 账本不存在或已归档：`LEDGER_NOT_FOUND`。
- 非账本成员访问：`LEDGER_ACCESS_DENIED`。
- 角色能力不足：`MEMBER_ROLE_DENIED`。
- 私密流水或私密账户不可见：`PRIVATE_RESOURCE_DENIED`。
- 账户不存在或不可见：`ACCOUNT_NOT_FOUND`。
- 分类类型不匹配、金额非法、时间缺少时区、转账目标账户非法、转账源账户与目标账户相同：`VALIDATION_FAILED`。
