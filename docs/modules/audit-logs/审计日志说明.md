# 审计日志说明

## 功能目标

审计日志模块记录敏感业务操作，帮助后台管理员排查问题，并为后续安全审计提供基础数据。

## 业务规则

- 审计日志只记录操作摘要，不记录密码、refresh token、票据图片原文或完整 AI prompt。
- 重要操作成功后写入审计日志；失败请求第一版暂不记录，后续可由异常过滤器统一扩展。
- 金额字段如进入 metadata，必须保持 Decimal 字符串，不使用 float。
- 审计日志不可通过普通业务接口修改或删除。
- M3 当前已实现基础写入能力，并通过 Admin API 提供系统管理员只读查询。
- M3 业务接入阶段会在账本、成员、账户、分类和流水的成功写操作后记录审计事件。

## 数据模型

`audit_logs`：

- `id`
- `actor_user_id`
- `ledger_id`
- `target_type`
- `target_id`
- `action`
- `summary`
- `metadata`
- `created_at`

## 接口说明

审计日志不暴露普通用户 HTTP 接口。

业务模块通过 `AuditLogsService.record()` 写入审计事件。调用方需要传入操作者、可选账本、目标类型、目标 ID、操作动作、摘要和可选 metadata。

系统管理员可通过 `GET /admin/audit-logs` 查询审计日志摘要。该接口只返回已脱敏 metadata，不提供修改或删除能力。

## 业务接入动作

M3 业务操作接入以下审计动作：

- `ledger.create`：创建账本。
- `ledger.update`：更新账本基础信息。
- `ledger.archive`：归档账本。
- `member.role.update`：更新账本成员角色。
- `member.remove`：移除账本成员。
- `account.create`：创建账户。
- `account.update`：更新账户基础信息或余额字段。
- `account.archive`：归档账户。
- `category.create`：创建分类。
- `category.update`：更新分类。
- `category.archive`：归档分类。
- `transaction.create`：创建正式流水。
- `transaction.update`：更新正式流水。
- `transaction.delete`：软删除正式流水。

审计 metadata 只记录排查所需的摘要字段，例如名称、类型、可见性、金额、币种、分类、账户或角色变化，不记录密码、token、票据原文、完整 AI prompt 或其他敏感原文。

## 权限规则

- 写入由业务 Service 触发，不暴露普通用户写入接口。
- 查询只允许系统管理员通过 Admin API 访问。
- 普通账本成员不能读取完整审计日志。

## 异常情况

- 审计日志写入失败不应绕过业务事务的一致性要求；接入具体业务操作时需按场景决定是否与主业务写入放在同一事务内。
- metadata 中的敏感字段会在 `AuditLogsService` 层过滤。
- 失败请求第一版不记录，后续可通过异常过滤器或请求日志扩展。

## 后续扩展点

- Admin API 审计日志查询后续可支持按操作者、账本、目标类型、动作和时间过滤。
- 后台写操作出现后，继续按独立设计文档接入审计日志。
- 增加失败请求、异常请求和高风险查看操作的审计记录。
