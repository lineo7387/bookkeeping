# 开发交接说明

## 新成员必读

1. `AGENTS.md`
2. `designer.md`
3. `docs/superpowers/specs/2026-05-17-bookkeeping-platform-design.md`
4. `.codex/project-context.md`
5. `.codex/development-rules.md`
6. `.codex/ai-service-guidelines.md`

## 开发前要求

- 先查看对应功能中文文档。
- 没有功能文档时，先补文档再写代码。
- 涉及前端页面时，先查看 `designer.md`。
- 涉及 AI 能力时，先查看 `.codex/ai-service-guidelines.md`。

## 脚手架协作

项目创建阶段由用户执行脚手架命令，Codex 负责创建后的配置和规范化。

第一批建议命令：

```bash
pnpm init
```

根 `package.json` 创建并检查后，再继续创建 NestJS：

```bash
pnpm create nest apps/api
```

后续命令按 `.codex/scaffolding-protocol.md` 执行。
