## 当前工程状态

- M0 已合并到 `main` 并由用户推送。
- M1 已合并到 `main`：Prisma 用户/会话/账本/成员/邀请模型、PrismaService、认证与会话、JWT Guard、Users/Auth/Ledgers/Policies 模块、Ledger Policy Guard、账本与成员管理 API、单元测试。
- M1.5 已合入并推送到 `main`：`accounts`、`categories`、`transactions` Prisma 模型与 NestJS API，补齐 `canViewAccount`、`canViewTransaction`、`canUpdateTransaction`，支持私密账户、私密流水、基础转账 metadata、软删除和对应单元测试。
- M2 账户余额流水联动已完成：正式流水创建、更新和软删除会在同一个 Prisma transaction 内调整相关账户 `current_balance`，支持收入、支出和单条转账模型；余额重算和余额变动台账保留为后续扩展。
- 基础统计 API 已完成首版：`GET /ledgers/:ledgerId/statistics/monthly`、`/categories`、`/accounts`、`/members` 支持账本成员 Policy 校验、私密流水过滤、私密账户过滤、Decimal 字符串输出和单元测试。
- 当前验证命令需以最近一次开发记录为准；基础统计开发完成时应重新运行：`pnpm --filter @bookkeeping/api test`、`pnpm --filter @bookkeeping/api typecheck`、`pnpm --filter @bookkeeping/api build`、`pnpm build`、`pnpm test`。

## 脚手架协作

项目创建阶段由用户执行脚手架命令，Codex 负责创建后的配置和规范化。

当前已完成根目录、`apps/api`、`apps/admin-web`、`packages/shared-types`、`packages/api-client`。`apps/api` 已进入 M1 业务基础能力阶段。后续脚手架计划以 `docs/handover/后续脚手架计划.md` 和 `.codex/scaffolding-protocol.md` 为准。

剩余重点顺序：

1. 继续 AI 编排、后台真实 API 接入或新脚手架前，应先补对应中文模块文档。
2. 视具体选题创建新的 `codex/` 前缀开发分支。
3. 用户执行 FastAPI `apps/ai-service` 创建命令后，Codex 再补内部服务配置和契约检查。
4. 用户确认 HBuilderX/CLI 流程后执行 uni-app `apps/mobile` 创建命令，Codex 再补移动端配置。
5. 后续根据重复度评估 `packages/validation` 和 `packages/config`。

关键边界：

- 前端、后台 Web 和移动端只调用 NestJS，不直接调用 FastAPI。
- AI 服务只生成候选结果，用户确认后才由 NestJS 创建正式流水。
- `api-client` 只能封装 NestJS 对外 API。
