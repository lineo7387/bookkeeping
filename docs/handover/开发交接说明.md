## 当前工程状态

- M0 已合并到 `main` 并由用户推送。
- M1 已合并到 `main`：Prisma 用户/会话/账本/成员/邀请模型、PrismaService、认证与会话、JWT Guard、Users/Auth/Ledgers/Policies 模块、Ledger Policy Guard、账本与成员管理 API、单元测试。
- M1.5 已合入并推送到 `main`：`accounts`、`categories`、`transactions` Prisma 模型与 NestJS API，补齐 `canViewAccount`、`canViewTransaction`、`canUpdateTransaction`，支持私密账户、私密流水、基础转账 metadata、软删除和对应单元测试。
- M2 账户余额流水联动已完成：正式流水创建、更新和软删除会在同一个 Prisma transaction 内调整相关账户 `current_balance`，支持收入、支出和单条转账模型；余额重算和余额变动台账保留为后续扩展。
- 基础统计 API 已完成首版：`GET /ledgers/:ledgerId/statistics/monthly`、`/categories`、`/accounts`、`/members` 支持账本成员 Policy 校验、私密流水过滤、私密账户过滤、Decimal 字符串输出和单元测试。
- M3 审计日志基础写入与业务接入能力已完成：新增 `audit_logs` Prisma 模型、`AuditLogsModule`、Repository、Service 和单元测试；`AuditLogsService.record()` 会过滤敏感 metadata；账本、成员、账户、分类、流水的成功写操作会写入审计日志。
- M3 后台只读 Admin API 基础能力已完成：新增 `users.is_system_admin`、`SystemAdminGuard`、`AdminModule`、用户列表、账本列表、AI 任务摘要列表和审计日志查询；所有 Admin 接口使用 JWT + 系统管理员 Guard。
- M3 后台 Web 真实 API 接入已完成首版，M4 已同步 AI 任务真实摘要文案：`apps/admin-web` 通过 `@bookkeeping/api-client` 调用 NestJS Admin API，首页展示用户、账本、AI 任务摘要和审计日志首屏分页样本，并提供加载、错误和刷新状态。后台 Web 不直接访问 FastAPI。
- M4 AI 文本记账 NestJS 闭环已完成首版：新增 `ai_tasks` / `ai_extractions` Prisma 模型、`AiModule`、内部 FastAPI client、文本解析任务、候选保存、候选确认/拒绝，确认后通过 `TransactionsService.createFromAiExtraction()` 创建 `source = ai_text` 的正式流水并复用账户余额联动逻辑；`GET /admin/ai/tasks` 已读取真实 AI 任务摘要。用户已使用 `uv` 创建 `apps/ai-service`，并已补充 FastAPI 内部文本解析契约 `POST /internal/ai/text-transaction` 的确定性 MVP parser、pytest 契约测试、本地运行示例和故障排查文档。本地启动验证已修复 Prisma 7 PostgreSQL adapter 和 AI internal client provider wiring 问题，FastAPI 使用 `uv run uvicorn app.main:app --host 127.0.0.1 --port 8000` 启动。后续继续实现时必须遵守：FastAPI 只返回候选结果，用户确认后才由 NestJS 创建正式流水。
- 当前验证命令需以最近一次开发记录为准；M4 AI 文本记账完成后应至少运行：`pnpm --filter @bookkeeping/api prisma:generate`、`pnpm --filter @bookkeeping/api test`、`pnpm --filter @bookkeeping/api typecheck`、`pnpm --filter @bookkeeping/api build`、`pnpm --filter @bookkeeping/shared-types build`、`pnpm --filter @bookkeeping/api-client test`、`pnpm --filter @bookkeeping/api-client build`、`pnpm --filter @bookkeeping/admin-web build`、`cd apps/ai-service && uv run pytest`、`pnpm build`、`pnpm test`、`git diff --check`。

## 开发前要求

- 先查看对应功能中文文档。
- 没有功能文档时，先补文档再写代码。
- 涉及前端页面时，先查看 `designer.md`。
- 涉及 AI 能力时，先查看 `.codex/ai-service-guidelines.md`。
- 涉及 NestJS 后端代码时，优先使用 `nestjs-best-practices`。
- 涉及 FastAPI AI 服务代码时，优先使用 `fastapi`。

## 脚手架协作

项目创建阶段由用户执行脚手架命令，Codex 负责创建后的配置和规范化。

当前已完成根目录、`apps/api`、`apps/admin-web`、`apps/ai-service`、`packages/shared-types`、`packages/api-client`。`apps/api` 已完成 M4 AI 文本记账 NestJS 闭环首版。后续脚手架计划以 `docs/handover/后续脚手架计划.md` 和 `.codex/scaffolding-protocol.md` 为准。

剩余重点顺序：

1. 当前分支 `codex/m4-ai-text-accounting` 用于继续 M4 AI 文本记账。
2. M4 FastAPI 文本解析内部契约和 NestJS 候选确认闭环已具备最小可测实现，本地运行和故障排查示例已补充；下一步可围绕真实端到端联调脚本/集成测试、低置信度候选补全或任务筛选继续扩展。
3. FastAPI 后续接入真实模型供应商前，必须保持当前内部响应契约和测试不破坏。
4. 用户确认 HBuilderX/CLI 流程后执行 uni-app `apps/mobile` 创建命令，Codex 再补移动端配置。
5. 后续根据重复度评估 `packages/validation` 和 `packages/config`。

关键边界：

- 前端、后台 Web 和移动端只调用 NestJS，不直接调用 FastAPI。
- AI 服务只生成候选结果，用户确认后才由 NestJS 创建正式流水。
- `api-client` 只能封装 NestJS 对外 API。
