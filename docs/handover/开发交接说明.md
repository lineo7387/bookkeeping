- 没有功能文档时，先补文档再写代码。
- 涉及前端页面时，先查看 `designer.md`。
- 涉及 AI 能力时，先查看 `.codex/ai-service-guidelines.md`。
- 涉及 NestJS 后端代码时，优先使用 `nestjs-best-practices`。
- 涉及 FastAPI AI 服务代码时，优先使用 `fastapi`。

## 脚手架协作

项目创建阶段由用户执行脚手架命令，Codex 负责创建后的配置和规范化。

当前已完成根目录、`apps/api`、`apps/admin-web`、`packages/shared-types`、`packages/api-client`。后续脚手架计划以 `docs/handover/后续脚手架计划.md` 和 `.codex/scaffolding-protocol.md` 为准。

剩余重点顺序：

1. 用户确认是否推送当前 `main` 或 `codex/m0-foundation-continuation`，Codex 不得未经批准执行 push。
2. 用户执行 FastAPI `apps/ai-service` 创建命令，Codex 再补内部服务配置和契约检查。
3. 用户确认 HBuilderX/CLI 流程后执行 uni-app `apps/mobile` 创建命令，Codex 再补移动端配置。
4. 后续根据重复度评估 `packages/validation` 和 `packages/config`。

关键边界：

- 前端、后台 Web 和移动端只调用 NestJS，不直接调用 FastAPI。
- AI 服务只生成候选结果，用户确认后才由 NestJS 创建正式流水。
- `api-client` 只能封装 NestJS 对外 API。
