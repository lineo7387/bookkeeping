# 接口规范

## 基本原则

- 所有前端只调用 NestJS API。
- API 字段使用英文。
- 错误信息面向用户时使用中文。
- 分页、错误、权限信息保持统一结构。

## 响应结构

```ts
type ApiResponse<T> = {
  success: boolean
  data?: T
  error?: {
    code: string
    message: string
    details?: unknown
  }
}
```

## 权限信息

账本相关接口应返回当前用户在账本中的角色和权限，方便 Web、小程序和 App 使用同一套控制逻辑。

```json
{
  "currentMember": {
    "role": "admin",
    "permissions": ["transaction:create", "member:invite"]
  }
}
```

## Admin AI 任务查询

`GET /admin/ai/tasks` 只返回脱敏 AI 任务摘要，可选 Query：

- `limit`：每页数量，默认 20，范围 1-100。
- `offset`：偏移量，默认 0。
- `status`：可选，`pending`、`processing`、`succeeded`、`failed`。
- `type`：可选，`text_parse`、`receipt_ocr`、`classify`、`insight`。

本接口不得返回 `inputText`、`rawResult` 或完整错误详情。
