# AI 服务规范

## 服务定位

FastAPI AI 服务只负责智能能力，不负责主业务状态，不直接创建正式交易流水。

NestJS 是唯一对外业务 API，负责认证、权限校验、账本成员校验、AI 任务创建、候选结果保存和确认后的正式流水创建。

## 第一阶段 AI 能力

- 文本记账解析：把自然语言转换为候选交易。
- 票据识别：从图片中识别金额、时间、商户和明细。
- 自动分类：根据文本、商户、历史习惯建议分类。

## 标准流程

```txt
用户输入或上传
  -> NestJS 校验权限
  -> NestJS 创建 ai_task
  -> NestJS 调用 FastAPI
  -> FastAPI 返回结构化候选结果
  -> NestJS 保存 ai_extraction
  -> 用户确认
  -> NestJS 创建 transaction
```

## 文本记账候选结果

```json
{
  "type": "expense",
  "amount": "86.00",
  "currency": "CNY",
  "occurredAt": "2026-05-17T12:00:00.000Z",
  "categoryName": "餐饮",
  "accountHint": "微信",
  "merchant": null,
  "note": "晚饭",
  "confidence": 0.91
}
```

## AI 结果约束

- 金额必须以字符串形式返回，避免浮点误差。
- 时间必须返回 ISO 8601 字符串。
- 不确定字段返回 `null`，不要编造。
- 必须返回 `confidence`。
- 原始模型响应应保存在 `raw_result`，方便排查和后续优化。
- 用户拒绝候选结果时，应保留拒绝状态，不能直接删除。

## 安全与隐私

- 不要把用户账本全量数据发送给模型。
- 只发送完成当前任务所需的最小上下文。
- 票据图片和识别结果属于敏感数据，应支持删除和过期清理。
- 日志中不要输出完整票据图片 URL、token、手机号、邮箱、密码摘要或 API Key。
